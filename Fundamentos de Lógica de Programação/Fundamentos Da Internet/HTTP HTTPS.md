# HTTP
HTTP (Hypertext Transfer Protocol) é o protocolo fundamental da internet para transferir dados, como páginas web (HTML), imagens e vídeos, entre um navegador (cliente) e um servidor, seguindo um modelo de solicitação e resposta, sendo a base para a World Wide Web e permitindo a navegação e troca de informações na rede. 

## Métodos

- GET
  - O método GET é uma requisição HTTP usada para solicitar e recuperar dados de um servidor, como uma página web, imagem ou arquivo JSON, sendo o mais comum para buscar informações, sem alterar seu estado, permitindo que os dados sejam enviados na URL e cacheáveis
- POST
  -  O método POST é uma requisição HTTP usada para enviar dados ao servidor para criar ou atualizar um recurso, como o envio de formulários, cadastros ou uploads de arquivos, com os dados enviados no corpo da mensagem, de forma oculta e sem restrição de tamanho, diferente do GET.
- PUT
  - O método PUT é um comando HTTP usado para criar ou substituir completamente um recurso em um servidor.
- DELETE
   - O método DELETE é um comando em computação para remover dados ou recursos

## Parâmetros

- Header
- Body
- Path

## Status

- 1xx - Informativo.
- 2xx - Sucesso.
- 3xx - Redirecionamento.
- 4xx - Erro Cliente.
- 5xx - Erro Servidor.

# HTTPS

HTTPS (HyperText Transfer Protocol Secure) é a versão segura do protocolo HTTP, que criptografa a comunicação entre seu navegador e um site, adicionando uma camada de segurança (SSL/TLS) para proteger dados sensíveis como senhas e informações de pagamento de interceptação, sendo identificado pelo cadeado na barra de endereço e sendo essencial para a confiança e segurança online.
